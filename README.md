# project-lens

A Claude Code plugin that prevents AI context loss on large codebases.

Uses a cheap OpenRouter LLM to scan your project and generate per-feature documentation. Before every file edit, the relevant doc + full file content is automatically injected into Claude's context — no skimming, no assumptions.

## The Problem

Claude skims large files. On unfamiliar code (like framework defaults you never wrote), it pattern-matches instead of reading, misses dependencies, and makes wrong assumptions that cause refactoring cycles.

## The Solution

```
You ask Claude to edit a file
        ↓
Hook fires automatically (PreToolUse)
        ↓
Full file content + feature doc injected into context
        ↓
Claude reads it — it's already there, nothing to skip
        ↓
Only then does Claude act
```

Feature docs are generated by a cheap LLM (DeepSeek, Gemma, etc.) via OpenRouter — not Claude tokens.

## Install

```bash
/plugin add github:0Serenvale/project-lens
```

Then configure your OpenRouter key:

```bash
claude plugin config project-lens openrouter_key YOUR_OPENROUTER_KEY
```

Optionally set a different model (default: `deepseek/deepseek-chat`):

```bash
claude plugin config project-lens model google/gemma-3-27b-it:free
```

## First Use (on any project)

```
/lens:init
```

This scans your entire project, generates `.lens/features/<feature>.md` for each detected feature, and creates a project overview. Run once per project.

## Commands

| Command | What it does |
|---|---|
| `/lens:init` | First-time full project scan |
| `/lens:search <topic>` | Load feature docs before starting a task |
| `/lens:scan <file>` | Rescan a specific file after changes |
| `/lens:update` | Rescan all files changed since last update |

## Recommended Workflow

```
Start task       → /lens:search <feature>   (load context first)
Make changes     → hook auto-injects file content before every edit
Code review      → /review
Commit           → git commit (hook auto-triggers rescan)
End of session   → /lens:update + update memory
```

## What Gets Generated

```
.lens/
  overview.md          ← project architecture summary
  index.md             ← file → feature mapping
  features/
    search.md          ← entry points, dependencies, data flow, gotchas
    auth.md
    matches.md
    ...
```

Each feature doc covers:
- **Entry points** — every exported function/component
- **Dependencies** — internal files, npm packages, config
- **Called by** — what imports this
- **Data flow** — step by step
- **Gotchas** — non-obvious behavior, edge cases, known issues
- **Status** — implemented / has issues / needs tests

## Supported Languages

TypeScript, JavaScript, TSX, JSX, Python, Go, Rust, PHP, Ruby, Java, Vue, Svelte

## Cost

Using `deepseek/deepseek-chat`: ~$0.0001 per file scan. A 50-file project costs less than $0.01 to initialize.

Free options: `google/gemma-3-27b-it:free`, `mistralai/mistral-7b-instruct:free`
